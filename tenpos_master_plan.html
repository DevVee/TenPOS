Role-Based Access Control (RBAC)
Architecture ▼
Super Admin
Full Access
  • Manage all users & roles
  • System-wide settings
  • All reports & analytics
  • Branch management
  • Audit log access
  • Price & discount control
Store Manager
Manager Access
  • View & export reports
  • Manage inventory
  • Manage cashier accounts
  • Apply discounts / voids
  • End-of-day reports
  • Cannot delete transactions
Cashier / Seller
Cashier Access
  • Process sales only
  • View own shift summary
  • Issue receipts
  • Apply pre-set discounts
  • Cannot access analytics
  • Cannot modify inventory
Read-Only Viewer
View Only
  • View reports & dashboards
  • Cannot process transactions
  • Cannot edit anything
  • Useful for owner/auditor
Implementation: Middleware + DB Level
Every API route checks JWT role claims. Supabase Row-Level Security (RLS) enforces access at the database itself — even if backend is bypassed, the DB rejects unauthorized reads/writes. Frontend routes are guarded by a ProtectedRoute HOC.

Core POS Features
Features ▼
🛒
Smart Sales Interface
Fast item search by name/barcode/SKU. Cart with quantity control, item notes, multi-payment split (cash + GCash + card). One-tap checkout. Supports numpad & barcode scanner input.
🧾
Receipt Management
Print via Bluetooth/USB thermal printer. Email or SMS digital receipt. QR code on receipt for transaction lookup. Customizable receipt header with Ten Foundation branding.
💳
Multiple Payment Methods
Cash (auto change computation), GCash/PayMaya (reference code logging), credit card, partial payment / layaway. Split payments across methods in one transaction.
↩️
Returns & Void
Manager-level void with reason logging. Full/partial returns. Automatic inventory restocking on return. Refund method tracks original payment type.
🏷️
Discounts & Promos
Percentage or fixed amount discounts. Senior/PWD discount (with ID logging). Promo codes. Manager-only override discounts. Time-based promos (e.g., weekend sale).
🔄
Offline Mode
Full POS functionality without internet. All transactions saved locally in WatermelonDB. Auto-sync queue uploads when connectivity is restored. Visual indicator shows sync status.

Integrated Inventory System
Features ▼
📦
Product & SKU Management
Add products with name, barcode, category, cost price, selling price, variants (size/color), images. Bulk import via CSV. Barcode printing for labels.
📉
Real-Time Stock Deduction
Every completed sale auto-deducts from inventory. Cancelled/voided transactions restock automatically. Stock levels reflect current inventory at all times.
⚠️
Low-Stock Alerts
Set reorder points per product. Dashboard alert and push notification when stock hits threshold. Export low-stock report for reordering.
📋
Stock Adjustments & Audit
Manual adjustment with reason (damage, theft, recount). Full audit log of who changed what and when. Physical count vs system count reconciliation.
🔢
Batch / Expiry Tracking
Track batch numbers and expiry dates per stock entry. Alert before expiry. FIFO (first in, first out) inventory method supported.
🏪
Multi-Location / Branch
Inventory tracked per branch. Stock transfer between branches with approval flow. Consolidated view for manager/admin. Suitable for future expansion.

Security Architecture (40+ Controls)
Security ▼
Authentication
S1
JWT + Short-lived Access Tokens
15-min access tokens, 7-day refresh tokens stored httpOnly cookie
S2
bcrypt Password Hashing
Cost factor 12+. Never store plaintext. Pepper added server-side.
S3
PIN Lock for Cashier
4-6 digit PIN to resume session. Auto-lock after idle timeout.
S4
Device Binding
POS terminals registered per device fingerprint. New device requires admin approval.
S5
Brute Force Protection
Rate limit login: 5 attempts → 15-min lockout. Account lockout with admin unlock.
Data & API Security
S6
Input Validation & Sanitization
Zod schema validation on all API inputs. Parameterized queries only (no raw SQL).
S7
HTTPS/TLS Everywhere
All API calls, sync endpoints, and dashboard traffic encrypted in transit.
S8
Local DB Encryption
WatermelonDB with SQLCipher for encrypted local storage on device.
S9
CORS + Helmet.js
Strict CORS policy. Security headers: CSP, X-Frame-Options, HSTS, etc.
S10
Row-Level Security (RLS)
PostgreSQL RLS policies ensure users can only read/write their authorized data.
Audit & Compliance
S11
Immutable Audit Log
Every action (sale, void, edit, login) logged with timestamp, user ID, IP. Logs are append-only.
S12
Transaction Integrity
Each transaction has a SHA-256 hash. Tampering is detectable. Backend double-checks totals.
S13
Role Enforcement at DB Layer
Even if frontend is compromised, DB policies enforce access. Defense in depth.
S14
Automatic Session Expiry
Cashier session locks after 5-min idle. Manager after 30-min. Full logout clears all tokens.

Anti-Vibe-Coding Security Checklist
Security ▼
  • ✕Never trust frontend data — validate ALL inputs server-side with Zod/Joi
  • ✕Never store JWT in localStorage — use httpOnly Secure cookies only
  • ✕Never compute totals on frontend only — backend re-validates every transaction total
  • ✕Never expose stack traces or DB errors to client — generic error messages only
  • ✕Never skip RBAC check — every API endpoint has explicit role guard, no implicit access
  • ✕Never use raw SQL string concatenation — parameterized queries enforced via ORM/query builder
  • ✕Never store secrets in code — all keys via environment variables, never committed to git
  • ✕Never allow mass assignment — whitelist allowed fields per endpoint explicitly
  • ✕Never skip rate limiting — all public endpoints rate-limited, auth endpoints extra strict
  • ✕Never skip audit logging — every sensitive action (void, price change, user creation) is logged immutably
